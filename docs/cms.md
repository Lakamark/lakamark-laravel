# CMS

## Overview

The platform includes an experimental CMS and dashboard system built with Laravel, Inertia.js and React.

The CMS architecture is currently under active development and may evolve significantly as the platform grows.

The goal is to provide a flexible foundation for content management, platform administration and future community-driven features.

---

## Current Features

Current CMS features include:

* Dashboard interface
* Content management
* Role-based access control
* Resource administration
* User management foundations

Some features are still incomplete or experimental.

---

## Architecture

The CMS follows a modular architecture designed to keep:

* Business logic separated
* Frontend interfaces maintainable
* Resources scalable
* Administrative tools extensible

The architecture is intentionally evolving during development.

---

## Frontend Stack

The CMS interface uses:

* Inertia.js
* React
* TypeScript
* Tailwind CSS
* Vite

These technologies are used only for authenticated management interfaces and not for the entire public platform.

---

## Goals

The CMS architecture aims to remain:

* Flexible
* Lightweight
* Modular
* Scalable
* Easy to extend

The long-term goal is to progressively build reusable management tools without introducing unnecessary complexity.
